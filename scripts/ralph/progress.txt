# Ralph Progress Log — Anchor Context Testing
Started: 2026-06-03

## Codebase Patterns
- Test directory: tests/
- Source directory: anchor-context/scripts/anchor/
- Python 3.9+, pytest 9.x
- Run tests: `python -m pytest tests/ -v`
- No external dependencies needed for testing (fallback mode)
- LLM judge tests require DEEPSEEK_API_KEY env var
- Hook scripts tested via subprocess with mock stdin
- Generated test data goes in tests/data/
- Use `mock.patch.dict(os.environ, {}, clear=True)` to force deterministic fallback mode when testing extract_graph (prevents LLM API key leakage from environment)

  - `time.perf_counter()` is preferred over `time.time()` for microbenchmarks (monotonic, higher resolution).

## 2026-06-04 09:28 - US-002
- What was implemented: Expanded entity extraction tests from 7 to 70 test items (63 new tests). 6 new test classes covering all acceptance criteria.
- Files changed: tests/test_core.py (+287 lines), prd.json (US-002 passes: true)
- **Learnings for future iterations:**
  - `_is_garbage()` (checks `_STOP_ENTITIES`) runs during `_extract_entities()`. `_is_proper_entity()` (checks `_common_word_blacklist`) runs later in `extract_graph()`. Tests at the `_extract_entities` level verify `_STOP_ENTITIES`-level filtering only.
  - `_RE_CMD_FLAG` matches `--flag` but `_is_garbage` filters anything starting with `-`. Cmd flags are effectively never extracted.
  - Chinese+English mixed text needs spaces between segments. Without spaces, `\b` boundaries fail at Chinese/ASCII transitions (both are `\w` in Unicode mode).
  - The Chinese regex `[一-鿿]{2,8}` is greedy (8-char max chunks). Test entity presence rather than specific substrings.
  - `_STOP_ENTITIES` has mixed case. `_is_garbage` checks `entity.lower()` → only entries with a lowercase variant get filtered by the `.lower()` path.
  - Test totals: 254 tests, 0.42s, all passing.

## 2026-06-04 09:45 - US-003
- What was implemented: Expanded bidirectional graph tests with 43 new tests across 6 test classes. Total test count now 297 (all passing).
- Files changed: tests/test_core.py (+312 lines: TestGraphVerbAnchors 10 tests, TestGraphNounAnchors 10 tests, TestGraphLinkIntegrity 8 tests, TestGraphDedup 5 tests, TestGraphTopN 5 tests, TestGraphEmptyInput 5 tests), prd.json (US-003 passes: true)
- **Learnings for future iterations:**
  - The `_extract_graph_fallback` helper uses `mock.patch.dict(os.environ, {}, clear=True)` to force deterministic fallback mode regardless of DEEPSEEK_API_KEY presence. This prevents LLM-dependent flakiness in CI.
  - `_GENERIC_VERBS` (set in extract_graph) filters common verbs like "use", "make", "need" — these will never appear as verb anchors even if VERB_MAP classifies them.
  - `_STOP_ENTITIES` includes "found" in lowercase → verbs extracted by segment_text may still be filtered in Phase 1 of extract_graph. Tests must use unfiltered DISCOVERY verbs like "identified", "discovered".
  - **Regex compound entity pitfall**: `_RE_PROPER_NAME` matches multi-word sequences like "Redis SETNX" and "Found JWT" as single entities. `_resolve_tags()` only does exact dict lookups, so compound entities won't get tags. Tests must use standalone entities for tag verification.
  - Verb anchors require at least one noun within 80-char window (`WINDOW_SIZE`) — messages with no extractable entities produce zero verb anchors.
  - Noun anchors require at least one non-generic verb within 80-char window — messages with only `_GENERIC_VERBS` produce zero noun anchors.
  - Fallback judge selection is deterministic: top 2 verbs + top 3 tech nouns + top 3 data nouns guaranteed (8 minimum), rest filled by score.
  - Test totals: 297 tests, 0.60s, all passing.

## 2026-06-04 10:45 - US-005
- What was implemented: 23 compression ratio benchmark tests across 5 test classes. All 349 tests pass (including existing).
- Files changed: tests/test_core.py (+560 lines: 5 new test classes), prd.json (US-005 passes: true)
- **Learnings for future iterations:**
  - Compression ratio formula: `(1 - graph.total_chars / total_message_chars) * 100`. `graph.total_chars` = sum(len(entity) + 10) per anchor — the +10 covers enum type/class serialization overhead.
  - With fallback mode (no API key), anchor count is capped at `max(8, n//2)` and the quota ensures top 2 verbs + top 3 tech nouns + top 3 data nouns = 8 minimum.
  - Backend/frontend message templates need ~80-120 chars each to hit 85% compression. Short messages (~50 chars) produce lower ratios because the anchor overhead is relatively larger.
  - Chinese text is inherently more compact (fewer chars per concept) — pure-Chinese conversations need ~75% threshold vs 80-85% for English.
  - Domain-specific templates should contain: decision verbs, anomalies, data values (versions/line-numbers/error-codes), and tech entities — this ensures the extractor has enough signal to work with.
  - `_extract_graph_fallback` helper (force-clear env) is essential for deterministic compression benchmarks — LLM mode adds non-determinism.
  - Test totals: 349 tests, 0.97s, all passing.
